Hexo博客搭建全过程记录

发表于 2016-03-02   |   分类于 前端   |  

写在前面的话

自从在github可以搭建静态博客后，我也加入了搭建个人博客的大潮。用过jekyll
，当时只是感到新潮，搭完之后也没怎么更新过东西就费掉了。用过csdn，cnblog等技术
园区的博客写文章，工作了之后也很少去更新。然而每每在浏览一些文章时，总能被感染
到，或是文章总结的有深度，或是页面做的精致令人赏心悦目。于是当我看到hexo可以如
此方便的搭建博客时，便忙里偷闲折腾了几天，就有了下面的一番折腾。。。

环境准备
  • 安装git
  • 安装node.js
  • 本人用的机器是win7 x64
hexo的安装和配置

hexo官网 hexo.io
运行git bash,
输入命令

$ npm install -g hexo

调试部署hexo

在我的电脑中建立一个文件夹，命名为Hexoblog，打开git bash，切换到此文件夹的路径
输入命令

$ hexo init

Hexo 会自动安装文件，等安装完成后，可以执行命令查看

然后在浏览器中输入localhost:4000 进行查看

更改主题

如果觉得默认的主题不够好，可以选择个人比较喜欢的主题进行替换
在git bash界面中切换到themes目录下，运行命令

$ git clone https://github.com/iissnan/hexo-theme-next themes/next

之后会在themes目录下生成文件夹next，可以查看next目录下的文件。另外，会在next下生成一个.git的隐藏文件。

启用主题

下载完主题后就是启用。可以参照官网 http://theme-next.iissnan.com/ 进行配置，官网描述的很清楚。
主要是修改Hexoblog目录下的config.yml配置文件中的theme属性，将其设置为next 就可以启用。更详细的配置直接参考官网。

$ hexo server -p 3000

配置SSH keys

通过ssh key的配置让本地git项目和远程github建立联系。

检查SSH keys的设置

首先检查电脑是否存在ssh key：

在回车中会提示你输入一个密码，这个密码会在你提交项目时使用，如果为空的话提交项目时则不用输入。这个设置是防止别人往你的项目里提交内容。
总之全部保持默认就行，一路回车会自动生成key

添加SSH Key到Github

在本机设置SSH Key之后，需要添加到Github上以完成SSH链接的设置。

  1. 打开本地C:\Users.ssh\id_rsa.pub文件。此文件里面内容为刚才生成的私钥。需要准确复制，如果没有则需要设置隐藏文件可见。
  1. 登陆github系统。点击右上角的 Account Settings—>SSH Public keys —> add another public keys
  1. 把本地生成的密钥复制到key文本框中， 点击 add key。
